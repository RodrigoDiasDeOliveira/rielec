RIELEC Documentation

Getting Started

Welcome to RIELEC, your advanced Docker Instance Management solution. This guide will help you get up and running quickly.

1. Sign Up

If you haven't already, sign up for a RIELEC account.

2. Connect Your Docker Environment

To connect your Docker environment to RIELEC, you'll need to install our agent. Run the following command in your terminal:

curl -sSL https://get.rielec.io | sudo bash
Note: Always verify the content of scripts before running them with sudo privileges.

3. Configure Your Dashboard

Once the agent is installed, log in to your RIELEC dashboard and follow the prompts to set up your first instance group.

Instance Management

RIELEC provides powerful tools for managing your Docker instances:

  • Create Instances: Use our intuitive interface to spin up new Docker instances with just a few clicks.
  • Instance Groups: Organize your instances into logical groups for easier management.
  • Scaling: Automatically scale your instances based on predefined rules or manually adjust as needed.

Monitoring

Stay informed about the health and performance of your Docker instances:

  • Real-time Metrics: Monitor CPU, memory, network, and disk usage in real-time.
  • Custom Alerts: Set up custom alerts based on performance thresholds or specific events.
  • Log Analysis: Centralized log collection and analysis for easier troubleshooting.

API Reference

RIELEC provides a comprehensive RESTful API for programmatic access to your Docker instances:

GET /api/v1/instances - List all instances POST /api/v1/instances - Create a new instance GET /api/v1/instances/{id} - Get details of a specific instance PUT /api/v1/instances/{id} - Update an instance DELETE /api/v1/instances/{id} - Delete an instance

For full API documentation, visit our API Documentation page.

Integrations

RIELEC integrates seamlessly with popular tools and platforms:

  • Kubernetes: Manage Kubernetes clusters alongside your Docker instances.
  • CI/CD Tools: Integrate with Jenkins, GitLab CI, and other popular CI/CD tools.
  • Cloud Providers: Native support for AWS, Azure, and Google Cloud Platform.

Troubleshooting

If you encounter any issues, try these steps:

  1. Check our Status Page for any ongoing service disruptions.
  2. Restart the RIELEC agent: sudo systemctl restart rielec-agent
  3. Review the agent logs: sudo journalctl -u rielec-agent
  4. If problems persist, contact our Support Team.
