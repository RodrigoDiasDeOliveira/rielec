RIELEC Full API Documentation

Introduction

Welcome to the comprehensive RIELEC API documentation. This API allows you to programmatically manage your Docker instances, monitor performance, and integrate RIELEC with your existing tools and workflows.

Base URL: https://api.rielec.io/v1

Authentication

All API requests must be authenticated using an API key. Include your API key in the Authorization header:

Authorization: Bearer YOUR_API_KEY

To obtain an API key, log in to your RIELEC dashboard and navigate to Settings > API Keys.

Instances

GET /instances

Retrieve a list of all instances.

Query Parameters:

ParameterTypeDescription
limitintegerNumber of instances to return (default: 20, max: 100)
offsetintegerOffset for pagination (default: 0)
statusstringFilter by instance status (e.g., running, stopped)

Response:

{ "instances": [ { "id": "inst-123abc", "name": "web-server-1", "status": "running", "image": "nginx:latest", "created_at": "2023-06-15T10:30:00Z", "resources": { "cpu": 1, "memory": 1024 } }, ... ], "total": 50, "limit": 20, "offset": 0 }

POST /instances

Create a new instance.

Request Body:

{ "name": "new-instance", "image": "nginx:latest", "resources": { "cpu": 1, "memory": 1024 }, "env": { "KEY1": "VALUE1", "KEY2": "VALUE2" }, "group_id": "grp-789ghi" // Optional }

Response:

{ "id": "inst-456def", "name": "new-instance", "status": "creating", "image": "nginx:latest", "created_at": "2023-06-16T14:20:00Z", "resources": { "cpu": 1, "memory": 1024 } }

GET /instances/{instance_id}

Retrieve details of a specific instance.

Response:

{ "id": "inst-123abc", "name": "web-server-1", "status": "running", "image": "nginx:latest", "created_at": "2023-06-15T10:30:00Z", "resources": { "cpu": 1, "memory": 1024 }, "env": { "KEY1": "VALUE1", "KEY2": "VALUE2" }, "group_id": "grp-789ghi" }

PUT /instances/{instance_id}

Update an existing instance.

Request Body:

{ "name": "updated-instance-name", "resources": { "cpu": 2, "memory": 2048 } }

Response:

{ "id": "inst-123abc", "name": "updated-instance-name", "status": "running", "image": "nginx:latest", "created_at": "2023-06-15T10:30:00Z", "updated_at": "2023-06-16T15:45:00Z", "resources": { "cpu": 2, "memory": 2048 } }

DELETE /instances/{instance_id}

Delete an instance.

Response:

{ "message": "Instance successfully deleted" }

Instance Groups

GET /instance-groups

Retrieve a list of all instance groups.

Response:

{ "groups": [ { "id": "grp-789ghi", "name": "web-servers", "instance_count": 5, "created_at": "2023-06-10T09:00:00Z" }, ... ] }

POST /instance-groups

Create a new instance group.

Request Body:

{ "name": "new-group", "description": "A group for new instances" }

Response:

{ "id": "grp-101jkl", "name": "new-group", "description": "A group for new instances", "instance_count": 0, "created_at": "2023-06-16T16:00:00Z" }

Monitoring

GET /instances/{instance_id}/metrics

Retrieve metrics for a specific instance.

Query Parameters:

ParameterTypeDescription
start_timestringStart time for metrics (ISO 8601 format)
end_timestringEnd time for metrics (ISO 8601 format)
metricstringComma-separated list of metrics (e.g., cpu,memory,network)

Response:

{ "instance_id": "inst-123abc", "metrics": { "cpu": [ {"timestamp": "2023-06-16T10:00:00Z", "value": 45.2}, {"timestamp": "2023-06-16T10:01:00Z", "value": 48.7}, ... ], "memory": [ {"timestamp": "2023-06-16T10:00:00Z", "value": 512.3}, {"timestamp": "2023-06-16T10:01:00Z", "value": 524.1}, ... ] } }

Users

GET /users

Retrieve a list of users (requires admin privileges).

Response:

{ "users": [ { "id": "usr-246abc", "email": "[email protected]", "role": "admin", "created_at": "2023-05-01T09:00:00Z" }, ... ] }

POST /users

Create a new user (requires admin privileges).

Request Body:

{ "email": "[email protected]", "password": "securepassword123", "role": "user" }

Response:

{ "id": "usr-357def", "email": "[email protected]", "role": "user", "created_at": "2023-06-16T17:30:00Z" }

Error Handling

The API uses conventional HTTP response codes to indicate the success or failure of an API request. In general:

  • 2xx range indicate success
  • 4xx range indicate an error that failed given the information provided (e.g., a required parameter was omitted, etc.)
  • 5xx range indicate an error with RIELEC's servers

Error Response Format:

{ "error": { "code": "error_code", "message": "A human-readable message providing more details about the error." } }

Rate Limiting

The RIELEC API implements rate limiting to ensure the stability and availability of the service for all users. The current limits are:

  • 1000 requests per hour for authenticated users
  • 50 requests per hour for unauthenticated requests

Rate limit information is included in the response headers:

X-RateLimit-Limit: 1000 X-RateLimit-Remaining: 999 X-RateLimit-Reset: 1623884461
